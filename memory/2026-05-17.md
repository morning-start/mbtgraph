**文件**: `src/storage/undirected_edge_list.mbt`

**与有向版 EdgeListGraph 的核心差异**:

| 方法 | 有向版 | 无向版 |
|------|--------|--------|
| `is_directed()` | `true` | **`false`** |
| `contains_edge(f,t)` | 仅 `f→t` | **双向匹配 `f↔t`** |
| `get_edge(f,t)` | 仅查 `from=f, to=t` | **双向查找** |
| `neighbors(id)` | 仅收集 `e.from == id` 的 `e.to` | **同时收集 `e.from` 和 `e.to`** |
| `degree(id)` | 仅计出边 | **计入边+出边总和** |

**Trait**: GraphReadable + GraphWritable + GraphEdgeIterable

### 差异 2：CSCGraph（压缩稀疏列格式）

**文件**: `src/storage/csc.mbt`

**CSR vs CSC 对比**:

| 维度 | CSR | CSC |
|------|-----|-----|
| 压缩维度 | 行（源节点/from）| 列（目标节点/to）|
| 主指针 | `row_ptr[i]` | `col_ptr[j]` |
| 副索引 | `col_idx[k]` → 目标 | `row_idx[k]` → 源 |
| 最优操作 | 出边遍历 O(deg_out) | **入边遍历 O(deg_in)** |
| 典型场景 | BFS/DFS/PageRank | **反向PageRank/入度查询** |

**两者均为只读格式**（LSP），通过 Builder 模式构建。

## 七、MoonBit 语法经验总结（重要！避免重复踩坑）

### 本次会话遇到的编译错误及解决方案

| # | 错误 | 原因 | 解决方案 |
|---|------|------|---------|
| 1 | `E3002: Unexpected { after Node` | MoonBit 结构体构造必须用 `::` | `Node{...}` → **`Node::{...}`** |
| 2 | `E4014: Expr Type Mismatch Float vs Double` | `.to_float()` 返回 Float 非 Double | 改用 **`.to_double()`** |
| 3 | `Parse error, unexpected token id` after `1e-300` | 科学计数法中 `-` 被解析为减法运算符 | 用 **`0.000001`** 替代 `1e-300` |
| 4 | `E3002: Parse error` in for 元组解构 | MoonBit 禁止 for 循环直接解构元组 | 先绑定再 match（AGENTS.md R5）|
| 5 | `assert_eq` 需要 Show trait | NodeId/Node/Edge/GraphError 未 derive Show | 用 **`assert_true(a == b)`** 替代 |

### 核心语法规则速查

```
结构体构造:    Type::{ field1: val1, field2: val2 }    ← 必须用 ::
Int→Double:    x.to_double()                           ← 非 to_float()
科学计数法:     避免使用 e-xx 格式                        ← 解析器问题
相等断言:      assert_true(a == b)                      ← 无 Show 时
Option匹配:    不需要 _ => () 分支                       ← AGENTS.md 推荐
Impl参数:      (self) 非 mut self                         ← AGENTS.md R2
```

### 过程中遇到的关键问题（重要经验！）

#### 问题 1: E4018 — 跨包类型不可见

**错误**: `Type DirectedAdjList does not implement trait @core.GraphReadable: no impl is defined`

**原因**: storage 包的 10 个 struct 使用 `pub struct`（包内可见），blackbox test 是独立包无法访问。

**修复**: 全部改为 `pub(all) struct`（涉及 10 个结构体）

#### 问题 2: E4063 — trait impl 可见性不一致

**错误**: `implementation of method edge_count is private`（同一 trait 内部分方法 pub、部分不 pub）

**原因**: MoonBit 要求 **trait 所有方法的 impl 可见性必须一致**。Agent 批量替换时只改了部分方法。

**修复**: ~144 个 `impl` 全部改为 `pub impl`，确保每个 trait 的所有方法统一为 pub。

**教训**: trait impl 可见性是**按 trait 粒度**统一控制的，不能对同一 trait 的不同方法设置不同可见性。

#### 问题 3: 🐛 Array::make 二维数组共享 bug

**现象**: matrix_test 中 in_degree 返回 50 而非 1，neighbors 长度异常

**根因**:
```moonbit
// ❌ 致命写法——所有行共享同一个内部数组
let matrix = Array::make(capacity, Array::make(capacity, None))
// Array::make(n, elem) 中 elem 只求值一次！
// 写入 matrix[0][1] = Some(5.0) 时，所有行的第1列都被污染
```

**修复**: 改为逐行独立创建：
```moonbit
let matrix : Array[Array[Double?]] = []
for _ in 0..<capacity {
  matrix.push(Array::make(capacity, None))
}
```

**影响文件**: directed_matrix.mbt + undirected_matrix.mbt

#### 问题 4: 测试假设与实现行为不一致（14 个失败）

| 失败数 | 原因 | 修复方式 |
|:------:|------|---------|

| 3 | CSR/CSC Builder 用 NodeId(5) 填充占位导致 contains_node 误判 | 改用从 NodeId(0) 开始的连续 ID |

### 规则体系

| 层级 | 可见性 | 谁能访问 | 典型场景 |
|------|--------|---------|---------|

## 十一、操作：转换器架构重构

### 问题背景

用户指出：转换器的 8 个函数全部使用 `[G : GraphReadable]` 约束，**混淆了两个独立维度**：
1. **存储结构转换**（AdjList ↔ Matrix ↔ CSR）— 合理，为算法优化
2. **有向/无向语义转换**（Directed ↔ Undirected）— 不应该由通用转换器隐式完成

### 改造方案（三层设计）

#### 第一层：有向图存储结构互转（编译期安全）

| 函数 | 约束 | 目标类型 |
|------|------|---------|
| `to_directed_adj_list(g)` | `[G : GraphDirected]` | DirectedAdjList |
| `to_directed_matrix(g, cap)` | `[G : GraphDirected]` | DirectedMatrix |
| `to_csr(g)` | `[G : GraphDirected]` | CSRGraph |
| `to_csc(g)` | `[G : GraphDirected]` | CSCGraph |
| `to_edge_list(g)` | `[G : GraphDirected]` | EdgeListGraph |

**保护机制**: 编译期保证——如果传入 UndirectedAdjList，直接报错 `does not implement trait GraphDirected`

#### 第二层：无向图存储结构互转（运行时保护）

| 函数 | 约束 | 目标类型 |
|------|------|---------|
| `to_undirected_adj_list(g)` | `[G : GraphReadable] raise` | UndirectedAdjList |
| `to_undirected_matrix(g, cap)` | `[G : GraphReadable] raise` | UndirectedMatrix |
| `to_undirected_edge_list(g)` | `[G : GraphReadable] raise` | UndirectedEdgeListGraph |

**保护机制**: 运行时断言 `assert_true(!is_directed(g))`，传入有向图会 panic

#### 第三层：显式语义转换（跨边界，用户知情）

| 函数 | 行为 |
|------|------|
| `as_undirected(directed_g)` | 有向→无向：`(a→b, w)` → `{a-b, w}`，双向边去重 |
| `as_directed(undirected_g)` | 无向→有向：`{a-b, w}` → `(a→b, w)`，不生成反向边 |

### 附带修复：CSR/CSC 补充 GraphDirected

CSR 和 CSC 是有向图格式但之前未实现 `GraphDirected` trait。本次补充各 6 个方法：

| 方法 | CSR 复杂度 | CSC 复杂度 | 原因 |
|------|-----------|-----------|------|
| in_neighbors / predecessors | O(E) 全扫描 | **O(deg_in)** 列遍历 | CSC 按列压缩 |
| out_neighbors / successors | **O(deg_out)** 行遍历 | O(E) 全扫描 | CSR 按行压缩 |
| in_degree | O(E) 计数 | **O(1)** 指针差 | CSC 天然支持 |
| out_degree | **O(1)** 指针差 | O(E) 计数 | CSR 天然支持 |
