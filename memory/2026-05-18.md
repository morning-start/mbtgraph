#### 1.3 Array::make 二维数组 Bug 修复

**问题**: `Array::make(n, Array::make(n, 0.0))` 共享同一行引用

**现象**: 修改 matrix[0][0] 影响 matrix[1][0], matrix[2][0] ...

**修复方案**:
```moonbit
// ❌ 错误：所有行共享同一数组引用
let matrix = Array::make(n, Array::make(n, 0.0))

// ✅ 正确：每行独立创建
let matrix : Array[Array[Double]] = []
for i in 0..<n {
  let row : Array[Double] = []
  for j in 0..<n {
    row.push(0.0)
  }
  matrix.push(row)
}
```

**影响范围**: directed_matrix.mbt / undirected_matrix.mbt / csr.mbt / csc.mbt

### 阶段 3: P1 最短路径模块 (13:30 - 15:30)

**设计决策**:
| 决策项 | 选择 | 原因 |
|--------|------|------|
| BinaryHeap | 自建私有实现 | MoonBit 标准库无堆结构 |
| 结果类型 | 独立 ShortestPathResult | 支持距离重建和路径查询 |
| Floyd-Warshall | 独立 FloydWarshallResult | 全源最短路径语义不同 |

#### 实现清单 ([shortest_path/](../src/algorithms/shortest_path/))

**核心数据结构**:

```moonbit
// 单源最短路径结果
pub(all) struct ShortestPathResult {
  distances : Array[Double?>       // None = 不可达
  parents : Array[@core.NodeId?]    // 路径重建
}
// 方法: distance_to() / is_reachable() / reachable_count() / path_to()

// 全源最短路径结果
pub(all) struct FloydWarshallResult {
  distances : Array[Array<Double?>>
  next : Array[Array[@core.NodeId?]>
}
// 方法: distance(from, to) / path(from, to)
```

**BinaryHeap 私有实现** ([heap.mbt](../src/algorithms/shortest_path/heap.mbt)):
- 可见性: `priv struct BinaryHeap`
- 存储元素: `(Double, @core.NodeId)` 元组，按 Double 升序
- 操作: push / pop / is_empty / length
- 算法: sift_up / sift_down（标准二叉堆）

**算法实现**:

| 文件 | 函数 | 复杂度 | 特性 |
|------|------|--------|------|
| [dijkstra.mbt](../src/algorithms/shortest_path/dijkstra.mbt) | `dijkstra(g, source)` | O((V+E)log V) | 非负权，自建堆优化 |
| [dijkstra.mbt](../src/algorithms/shortest_path/dijkstra.mbt) | `dijkstra_targeted(g, src, tgt)` | O((V+E)log V) | 提前终止优化版 |
| [bellman_ford.mbt](../src/algorithms/shortest_path/bellman_ford.mbt) | `bellman_ford(g, source)` | O(VE) | 支持负权边，检测负环 |
| [floyd_warshall.mbt](../src/algorithms/shortest_path/floyd_warshall.mbt) | `floyd_warshall(g)` | O(V³) | 全源最短路径 |

## 💡 关键经验提炼

### 技术层面

1. **MoonBit 嵌套函数限制**
   - 问题: Tarjan 的 `strongconnect` 无法定义在 `tarjan_scc` 内部
   - 方案: 提取为包级 `priv fn`，用 `TarjanState` 结构体携带可变状态
   - 模式: 单元素数组 `[value]` 模拟可变标量

2. **MoonBit 栈实现限制**
   - 问题: Kosaraju 第二次 DFS 无法用 `array_pop_last`（let 不能重绑定）
   - 方案: 预分配 `stack_data` 数组 + `stack_top` 整数指针模拟栈

3. **Trait 约束升级**
   - 场景: Kosaraju 需要 `in_neighbors`（反向邻接查询）
   - 变更: `fn[G : GraphReadable]` → `fn[G : GraphReadable + GraphDirected]`
   - 副作用: 必须为 EdgeListGraph 补充 GraphDirected 实现

4. **`let g = add_node(...) |> ignore` 反模式**
   - 问题: 返回值被 rebind 到 Unit，原变量失效
   - 修复: 直接调用 `add_node(g, x) |> ignore`（不重新绑定）
   - 出现频率: **每个测试文件都遇到**（generators/shortest_path/mst/connectivity）

### 工程层面

5. **设计先行策略**
   - 流程: brainstorming → 用户确认技术选型 → 写 design doc → 实现
   - 优势: 减少返工，确保 API 设计合理

6. **分层测试策略**
   - 基础功能: 类型创建/方法正确性（~30%）
   - 算法正确性: 经典案例/已知答案（~40%）
   - 边界情况: 空图/越界/异常输入（~20%）
   - 属性验证: 不可变性/一致性约束（~10%）

7. **跨存储兼容性测试**
   - 策略: 每个算法至少在 3 种存储上测试（AdjList/Matrix/EdgeList）
   - 目的: 验证 trait 约束的通用性
