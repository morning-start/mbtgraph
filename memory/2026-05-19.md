## ⏰ 操作时间线

### 09:00 - 10:30 | 修复 141 个编译错误

**背景**: flow 模块代码已创建但编译失败（141 errors）

**问题诊断**:
```
❌ Array[Array[Double?>>     → 嵌套泛型 >> 冲突
❌ Array<Double>             → 同上 > 问题
❌ let fn = self             → fn 是保留字
❌ var x = 0                 → 废弃语法
❌ loop { }                  → 废弃语法
```

**修复方案**:
1. `Double?>>` → `Double?]]` （避免位移符冲突）
2. `fn` → `net` / `graph` （避开保留字）
3. `var` → `let mut`
4. `loop {}` → `while true {}`
5. 移除有问题的 `cur == parent[cur]` 类型检查

**结果**: ✅ **141 errors → 0 errors** (moon check 通过)

## 💡 关键经验提炼

### 技术层面

1. **MoonBit 嵌套泛型陷阱**: `>>` 被解析为位移符，必须使用 `]]`
2. **保留字冲突**: `fn`/`var` 不能作变量名，改用语义化名称
3. **纯函数语义**: 算法修改输入数据时必须深拷贝（数组是引用类型）
4. **返回值消费**: MoonBit 不允许隐式丢弃非 Unit 返回值

### 工程层面

5. **分部提交策略**: 类型→算法→测试→文档（底层→上层）保证可追溯性
6. **Conventional Commits**: 统一格式便于自动化工具处理
7. **文档标准化**: 8 大章节模板确保完整性（API/示例/原理/内部/边界/测试/决策/配合）

### 流程层面

8. **先修编译再写测试**: 避免浪费时间调试语法错误
9. **边界情况优先**: 测试时先覆盖异常路径再验证正常逻辑
10. **经验固化机制**: 每次开发后立即更新 AGENTS.md 避免重复踩坑
