# 图数据存储方式调研分析

> **版本**: v0.1.0 | **状态**: 草稿 | **日期**: 2026-05-07

---

## 1. 概述

图是一种重要的数据结构，用于表示对象之间的关系。选择合适的图存储方式对算法性能和内存效率至关重要。本报告系统调研了8种主流的图数据存储方式，分析其核心特点、复杂度、适用场景及优缺点。

---

## 2. 存储方式详细分析

### 2.1 邻接矩阵 (Adjacency Matrix)

#### 核心特点

```
┌─────────────────────────────────────────────────────────┐
│                    邻接矩阵示意                          │
├─────────────────────────────────────────────────────────┤
│                    节点索引 (j)                          │
│      │ 0 │ 1 │ 2 │ 3 │ 4 │                            │
│ ─────┼───┼───┼───┼───┼───┤                            │
│  0   │ 0 │ 1 │ 0 │ 1 │ 0 │  ← 节点0与节点1、3相连      │
│  1   │ 1 │ 0 │ 1 │ 0 │ 0 │  ← 节点1与节点0、2相连      │
│(i)  2 │ 0 │ 1 │ 0 │ 1 │ 1 │  ← 节点2与节点1、3、4相连   │
│  3   │ 1 │ 0 │ 1 │ 0 │ 0 │                            │
│  4   │ 0 │ 0 │ 1 │ 0 │ 0 │                            │
└─────────────────────────────────────────────────────────┘
```

#### 数据结构定义

```moonbit
// 邻接矩阵存储结构
struct AdjacencyMatrix[T] {
  size : Int                    // 节点数量
  matrix : Array[Array[Bool]]   // 布尔矩阵: matrix[i][j] 表示边 (i,j) 是否存在
}

// 带权重的邻接矩阵
struct WeightedAdjacencyMatrix[T, W] {
  size : Int
  matrix : Array[Array[Option[W]]]  // None 表示无边
}
```

#### 复杂度分析

| 操作 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 创建 | O(V²) | O(V²) |
| 查询边 (i,j) | O(1) | - |
| 添加边 | O(1) | - |
| 删除边 | O(1) | - |
| 遍历邻接点 | O(V) | - |

#### 适用场景

- **最佳场景**: 小而稠密的图 (E ≈ V²)
- **典型应用**: 
  - 社交网络的小团体分析
  - 电路设计中的连接矩阵
  - 矩阵运算优化的图算法

#### 优缺点

| 优点 | 缺点 |
|------|------|
| 查询连通性极快 (O(1)) | 空间复杂度高 (O(V²)) |
| 实现简单直观 | 稀疏图浪费大量空间 |
| 适合矩阵运算优化 | 动态增减节点困难 |

### 2.2 邻接表 (Adjacency List)

#### 核心特点

```
┌─────────────────────────────────────────────────────────┐
│                    邻接表示意                            │
├─────────────────────────────────────────────────────────┤
│  节点0: [1] → [3] → NULL                                │
│  节点1: [0] → [2] → NULL                                │
│  节点2: [1] → [3] → [4] → NULL                         │
│  节点3: [0] → [2] → NULL                                │
│  节点4: [2] → NULL                                      │
└─────────────────────────────────────────────────────────┘
```

#### 数据结构定义

```moonbit
// 邻接表存储结构
struct AdjacencyList[T] {
  nodes : Array[Node[T]]
  adj : Array[Array[NodeId]]   // adj[i] 存储节点i的所有邻居
}

// 带权重的邻接表
struct WeightedAdjacencyList[T, W] {
  nodes : Array[Node[T]]
  adj : Array[Array[(NodeId, W)]]  // (目标节点, 权重)
}
```

#### 复杂度分析

| 操作 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 创建 | O(V+E) | O(V+E) |
| 查询边 (i,j) | O(deg(i)) | - |
| 添加边 | O(1) | - |
| 删除边 | O(deg(i)) | - |
| 遍历邻接点 | O(deg(i)) | - |

#### 适用场景

- **最佳场景**: 通用稀疏图 (E ≈ V ~ V log V)
- **典型应用**:
  - 社交网络 (平均度数 ~ 100)
  - 网页链接图
  - 大多数图算法的默认存储

#### 优缺点

| 优点 | 缺点 |
|------|------|
| 空间效率高 (O(V+E)) | 查询边存在性较慢 |
| 动态增删节点方便 | 内存局部性较差 |
| 适合大多数通用场景 | 缓存命中率低 |

### 2.3 边集数组 (Edge List)

#### 核心特点

```
┌─────────────────────────────────────────────────────────┐
│                    边集数组示意                          │
├─────────────────────────────────────────────────────────┤
│  边索引 |  起点 |  终点 |  权重 (可选)                   │
│  ──────|───────|───────|───────────                      │
│    0   |   0   |   1   |    5.0                         │
│    1   |   0   |   3   |    3.0                         │
│    2   |   1   |   2   |    2.0                         │
│    3   |   2   |   3   |    4.0                         │
│    4   |   2   |   4   |    1.0                         │
└─────────────────────────────────────────────────────────┘
```

#### 数据结构定义

```moonbit
// 边集数组存储结构
struct EdgeList[T] {
  nodes : Array[Node[T]]
  edges : Array[Edge]      // Edge = { from: NodeId, to: NodeId, weight?: W }
}
```

#### 复杂度分析

| 操作 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 创建 | O(E) | O(V+E) |
| 查询边 (i,j) | O(E) | - |
| 添加边 | O(1) | - |
| 删除边 | O(E) | - |
| 遍历邻接点 | O(E) | - |

#### 适用场景

- **最佳场景**: Kruskal 算法、边遍历优先的场景
- **典型应用**:
  - 最小生成树 (Kruskal)
  - 批量边处理
  - 图的序列化/持久化

#### 优缺点

| 优点 | 缺点 |
|------|------|
| 存储简洁，适合排序 | 查询效率低 |
| 边操作高效 | 不支持快速邻接查询 |
| 内存连续，缓存友好 | 需要额外索引支持快速查询 |

### 2.4 十字链表 (Orthogonal List)

#### 核心特点

```
┌─────────────────────────────────────────────────────────┐
│                    十字链表示意                          │
├─────────────────────────────────────────────────────────┤
│  每个边节点包含:                                         │
│    ┌─────────────────────────────────────┐              │
│    │ tail  → 起点节点                      │              │
│    │ head  → 终点节点                      │              │
│    │ hlink → 同起点的下一条边               │              │
│    │ tlink → 同终点的下一条边               │              │
│    └─────────────────────────────────────┘              │
├─────────────────────────────────────────────────────────┤
│  节点0 出边链: [0→1] → [0→3] → NULL                    │
│  节点0 入边链: [1→0] → NULL                            │
│                                                        │
│  节点1 出边链: [1→2] → NULL                            │
│  节点1 入边链: [0→1] → NULL                            │
└─────────────────────────────────────────────────────────┘
```

#### 数据结构定义

```moonbit
// 十字链表边节点
struct OrthogonalEdge[W] {
  tail : NodeId              // 起点
  head : NodeId              // 终点
  weight : Option[W]         // 权重
  hlink : Option[EdgeRef]    // 同起点的下一条边
  tlink : Option[EdgeRef]    // 同终点的下一条边
}

// 十字链表存储结构
struct OrthogonalList[T, W] {
  nodes : Array[Node[T]]
  first_in : Array[Option[EdgeRef]]   // 各节点的第一条入边
  first_out : Array[Option[EdgeRef]]  // 各节点的第一条出边
  edges : Array[OrthogonalEdge[W]]
}
```

#### 复杂度分析

| 操作 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 创建 | O(V+E) | O(V+E) |
| 查询边 (i,j) | O(min(indeg(i), outdeg(j))) | - |
| 添加边 | O(1) | - |
| 删除边 | O(deg) | - |
| 遍历邻接点 | O(deg) | - |

#### 适用场景

- **最佳场景**: 有向图、依赖拓扑分析
- **典型应用**:
  - 程序依赖图分析
  - 工作流/任务调度图
  - 需要同时访问入边和出边的场景

#### 优缺点

| 优点 | 缺点 |
|------|------|
| 同时支持入边和出边快速遍历 | 实现复杂 |
| 适合有向图操作 | 内存开销较大 |
| 删除边效率较高 | 缓存局部性差 |

### 2.5 邻接多重表 (Adjacency Multilist)

#### 核心特点

```
┌─────────────────────────────────────────────────────────┐
│                    邻接多重表示意                        │
├─────────────────────────────────────────────────────────┤
│  无向边只存储一次，通过指针关联两个顶点                   │
│                                                        │
│  边节点结构:                                            │
│    ┌─────────────────────────────────────┐              │
│    │ mark   → 删除标记                      │              │
│    │ vertex1 → 顶点1                        │              │
│    │ vertex2 → 顶点2                        │              │
│    │ path1   → 顶点1的下一条边              │              │
│    │ path2   → 顶点2的下一条边              │              │
│    └─────────────────────────────────────┘              │
│                                                        │
│  节点0: [0-1] → [0-3] → NULL                          │
│  节点1: [0-1] → [1-2] → NULL                          │
│  节点2: [1-2] → [2-3] → [2-4] → NULL                   │
└─────────────────────────────────────────────────────────┘
```

#### 数据结构定义

```moonbit
// 邻接多重表边节点
struct MultiEdge[W] {
  mark : Bool                // 删除标记
  vertex1 : NodeId           // 顶点1
  vertex2 : NodeId           // 顶点2
  weight : Option[W]         // 权重
  path1 : Option[EdgeRef]    // 顶点1的下一条边
  path2 : Option[EdgeRef]    // 顶点2的下一条边
}

// 邻接多重表存储结构
struct AdjacencyMultilist[T, W] {
  nodes : Array[Node[T]]
  adj : Array[Option[EdgeRef]]   // 各节点的第一条边
  edges : Array[MultiEdge[W]]
}
```

#### 复杂度分析

| 操作 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 创建 | O(V+E) | O(V+E) |
| 查询边 (i,j) | O(min(deg(i), deg(j))) | - |
| 添加边 | O(1) | - |
| 删除边 | O(deg) | - |
| 遍历邻接点 | O(deg) | - |

#### 适用场景

- **最佳场景**: 无向大图、路网
- **典型应用**:
  - 道路网络
  - 社交网络 (无向关系)
  - 需要高效处理无向边的场景

#### 优缺点

| 优点 | 缺点 |
|------|------|
| 无向边只存一次，节省空间 | 实现较复杂 |
| 边操作效率高 | 有向图不适用 |
| 删除边只需标记 | 指针管理复杂 |

### 2.6 链式前向星 (Forward Star)

#### 核心特点

```
┌─────────────────────────────────────────────────────────┐
│                    链式前向星示意                        │
├─────────────────────────────────────────────────────────┤
│  使用数组模拟邻接表，无指针操作                          │
│                                                        │
│  head 数组: head[u] = 节点u的第一条边索引               │
│  edge 数组: edge[i] = (to, weight, next)               │
│                                                        │
│  head: [0, 2, 4, 7, 9, -1]                            │
│                                                        │
│  edge[0]: to=1, w=5, next=-1   ← 边 0→1               │
│  edge[1]: to=3, w=3, next=-1   ← 边 0→3               │
│  edge[2]: to=2, w=2, next=-1   ← 边 1→2               │
│  edge[3]: to=0, w=5, next=-1   ← 边 1→0               │
└─────────────────────────────────────────────────────────┘
```

#### 数据结构定义

```moonbit
// 链式前向星边结构
struct ForwardStarEdge[W] {
  to : NodeId              // 目标节点
  weight : W               // 权重
  next : Int               // 下一条边的索引 (-1 表示结束)
}

// 链式前向星存储结构
struct ForwardStar[T, W] {
  nodes : Array[Node[T]]
  head : Array[Int]           // head[u] = 节点u的第一条边索引
  edges : Array[ForwardStarEdge[W]]
}
```

#### 复杂度分析

| 操作 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 创建 | O(V+E) | O(V+E) |
| 查询边 (i,j) | O(outdeg(i)) | - |
| 添加边 | O(1) | - |
| 删除边 | O(outdeg(i)) | - |
| 遍历邻接点 | O(outdeg(i)) | - |

#### 适用场景

- **最佳场景**: 算法竞赛、高性能图算法
- **典型应用**:
  - 图论算法竞赛
  - 需要极致性能的场景
  - 嵌入式/资源受限环境

#### 优缺点

| 优点 | 缺点 |
|------|------|
| 极致性能，无指针开销 | 不支持快速入边遍历 |
| 内存连续，缓存友好 | 动态删除困难 |
| 实现简洁高效 | 需要预分配空间 |

### 2.7 关联矩阵 (Incidence Matrix)

#### 核心特点

```
┌─────────────────────────────────────────────────────────┐
│                    关联矩阵示意                          │
├─────────────────────────────────────────────────────────┤
│                    边索引 (j)                          │
│      │ 0 │ 1 │ 2 │ 3 │ 4 │                            │
│ ─────┼───┼───┼───┼───┼───┤                            │
│  0   │ 1 │ 1 │ 0 │ 0 │ 0 │  ← 节点0关联边0、边1        │
│  1   │-1 │ 0 │ 1 │ 0 │ 0 │  ← 节点1关联边0、边2        │
│(i)  2 │ 0 │ 0 │-1 │ 1 │ 1 │  ← 节点2关联边2、边3、边4   │
│  3   │ 0 │-1 │ 0 │-1 │ 0 │                            │
│  4   │ 0 │ 0 │ 0 │ 0 │-1 │                            │
├─────────────────────────────────────────────────────────┤
│  注: 1=起点, -1=终点, 0=无关                           │
└─────────────────────────────────────────────────────────┘
```

#### 数据结构定义

```moonbit
// 关联矩阵存储结构
struct IncidenceMatrix[T] {
  nodes : Array[Node[T]]
  edges : Array[Edge]
  matrix : Array[Array[Int]]   // 1=起点, -1=终点, 0=无关
}
```

#### 复杂度分析

| 操作 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 创建 | O(V×E) | O(V×E) |
| 查询边关联 | O(V) | - |
| 添加边 | O(V) | - |
| 删除边 | O(V) | - |
| 遍历邻接点 | O(E) | - |

#### 适用场景

- **最佳场景**: 数学建模、网络流理论
- **典型应用**:
  - 图论数学证明
  - 矩阵运算分析
  - 网络流问题建模

#### 优缺点

| 优点 | 缺点 |
|------|------|
| 数学表达清晰 | 空间复杂度高 (O(V×E)) |
| 适合矩阵运算 | 实际应用效率低 |
| 便于理论分析 | 不适合大规模图 |

### 2.8 CSR/CSC (Compressed Sparse Row/Column)

#### 核心特点

```
┌─────────────────────────────────────────────────────────┐
│                    CSR 格式示意                          │
├─────────────────────────────────────────────────────────┤
│  row_ptr: 每行起始位置                                   │
│  col_idx: 列索引                                        │
│  values:  非零元素值 (权重)                              │
│                                                        │
│  row_ptr: [0, 2, 4, 7, 9, 9]                          │
│  col_idx: [1, 3, 0, 2, 1, 3, 4, 0, 2]                 │
│  values:  [5, 3, 5, 2, 2, 4, 1, 3, 4]                 │
│                                                        │
│  解释:                                                  │
│    row 0: cols [1,3], values [5,3]                     │
│    row 1: cols [0,2], values [5,2]                     │
│    row 2: cols [1,3,4], values [2,4,1]                 │
└─────────────────────────────────────────────────────────┘
```

#### 数据结构定义

```moonbit
// CSR 格式存储结构
struct CSR[T, W] {
  nodes : Array[Node[T]]
  row_ptr : Array[Int]       // 每行起始位置
  col_idx : Array[NodeId]    // 列索引 (目标节点)
  values : Array[W]          // 权重值
}

// CSC 格式存储结构
struct CSC[T, W] {
  nodes : Array[Node[T]]
  col_ptr : Array[Int]       // 每列起始位置
  row_idx : Array[NodeId]    // 行索引 (源节点)
  values : Array[W]          // 权重值
}
```

#### 复杂度分析

| 操作 | 时间复杂度 | 空间复杂度 |
|------|-----------|-----------|
| 创建 | O(V+E) | O(V+E) |
| 查询边 (i,j) | O(log deg(i)) | - |
| 添加边 | O(E) | - |
| 删除边 | O(E) | - |
| 遍历邻接点 | O(deg(i)) | - |

#### 适用场景

- **最佳场景**: 亿级大图、GNN、图计算
- **典型应用**:
  - 大规模图神经网络训练
  - 分布式图计算
  - 数据库索引

#### 优缺点

| 优点 | 缺点 |
|------|------|
| 极致内存效率 | 不支持动态修改 |
| 缓存局部性极佳 | 构建成本高 |
| 适合批处理操作 | 查询边存在性慢 |

## 4. 选型建议

### 4.1 按图规模选择

```
            图规模 →
            小 (V<1K)   |   中 (V<1M)    |   大 (V>1M)
┌─────────────────────────────────────────────────────────┐
稠密图 (E≈V²)  │  邻接矩阵     |   邻接矩阵     |   CSR/CSC
├─────────────────────────────────────────────────────────┤
稀疏图 (E≈V)   │  邻接表/链式前向星 |   邻接表/CSR   |   CSR/CSC
├─────────────────────────────────────────────────────────┤
动态图         │  邻接表       |   邻接表       |   专用图数据库
```

### 4.2 按算法需求选择

| 算法类型 | 推荐存储 | 理由 |
|----------|----------|------|
| BFS/DFS | 邻接表、链式前向星 | 需要高效遍历邻居 |
| Dijkstra | 邻接表、CSR | 快速访问出边 |
| Kruskal | 边集数组 | 需要边排序 |
| Prim | 邻接表、CSR | 需要高效权值访问 |
| PageRank | CSR | 内存效率优先 |
| 社区检测 | CSR、邻接表 | 批量处理友好 |
| 网络流 | 邻接表、CSR | 残差图操作 |

### 4.3 mbtgraph 库选型决策

基于上述分析，`mbtgraph` 库将采用以下策略：

| 场景 | 实现方案 |
|------|----------|
| **默认实现** | 邻接表 (`AdjacencyList`) |
| **稠密图优化** | 邻接矩阵 (`AdjacencyMatrix`) |
| **大规模只读** | CSR 格式 (`CSRGraph`) |
| **动态图** | 邻接表 + HashMap |

## 5. 总结

| 存储方式 | 最佳适用场景 | 关键优势 |
|----------|--------------|----------|
| **邻接矩阵** | 小稠密图 | O(1) 查询 |
| **邻接表** | 通用稀疏图 | 空间效率 + 灵活性 |
| **边集数组** | Kruskal、边遍历 | 边排序友好 |
| **十字链表** | 有向图、依赖分析 | 出入边双向遍历 |
| **邻接多重表** | 无向大图、路网 | 无向边存储优化 |
| **链式前向星** | 算法竞赛、高性能 | 极致性能 |
| **关联矩阵** | 数学建模、网络流 | 矩阵运算友好 |
| **CSR/CSC** | 亿级大图、GNN | 极致内存效率 |
