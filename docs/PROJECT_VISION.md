## 🎯 一句话定义

**mbtgraph 是 MoonBit 生态系统中首个生产级图算法库，提供从基础数据结构到高级图分析的完整解决方案。**

---

## 🌟 为什么需要 mbtgraph？

### MoonBit 生态的空白

MoonBit 作为新兴的多后端编程语言（支持 native/wasm/js），在 2026 年仍面临一个关键问题：

```
┌─────────────────────────────────────────────────────┐
│                MoonBit 生态系统 (2026)               │
│                                                      │
│  ✅ Web 框架    ✅ 数据库驱动   ✅ HTTP 客户端        │
│  ✅ JSON 解析   ✅ 日志库       ✅ 测试框架            │
│  ❌ 图算法库    ❌ 图数据结构   ❌ 网络分析工具        │  ← 空白！
│                                                      │
└─────────────────────────────────────────────────────┘
```

**现状**: 开发者如果需要在 MoonBit 中使用图算法，必须：
1. 自己从零实现（重复造轮子）
2. 通过 FFI 调用其他语言的库（性能损失 + 复杂度）
3. 放弃图相关的功能需求（限制应用场景）

**mbtgraph 的使命**: 填补这一空白，让 MoonBit 开发者开箱即用地使用高质量图算法。

## 💡 项目价值主张

### 对 MoonBit 开发者

| 场景 | 没有 mbtgraph | 有 mbtgraph |
|------|:-------------:|:-----------:|
| **社交网络分析** | 需要自己实现 PageRank/社区检测 | `pagerank(graph)` 一行调用 |
| **路径规划** | 手写 Dijkstra/A* (100+ 行 bug 多) | `dijkstra(graph, source)` 生产级实现 |
| **网络流优化** | 无法实现 | `max_flow(network, s, t)` 直接可用 |
| **推荐系统** | 缺少图匹配算法 | `hungarian(cost_matrix)` 二分图匹配 |
| **Wasm 前端** | 无图可视化数据源 | DOT/JSON 序列化直接对接 D3.js |

### 对 MoonBit 生态

| 维度 | 贡献 |
|------|------|
| **生态完整性** | 补齐"数据结构与算法"版图的关键拼图 |
| **语言竞争力** | 展示 MoonBit 在系统编程领域的能力 |
| **学习资源** | 为学习者提供算法实现的 MoonBit 最佳实践 |
| **工业就绪** | 证明 MoonBit 可用于生产级数学计算 |

### 技术创新价值

| 创新点 | 说明 |
|--------|------|
| **6 层 Trait 设计** | 比 petgraph(3层)/JGraphT(4层) 更细粒度的抽象 |
| **8 种存储结构** | 覆盖所有主流图存储方式，比竞品更全面 |
| **纯函数语义** | MoonBit 不可变性保证，天然适合并行计算 |
| **多后端统一** | 同一套代码编译到 native/wasm/js，无需修改 |

## 📊 项目定位与差异化

### 在图算法库领域的位置

```
                    高级功能 (ML/GNN)
                    ↑
                    │    NetworkX (Python)
                    │    (功能最全，性能一般)
                    │
   功能丰富度  │         mbtgraph (MoonBit)
              │         (🎯 平衡点：功能+性能+体积)
              │              petgraph (Rust)
              │              (高性能，功能较少)
              │
              └──────────────────────────→ 性能
                 LEMON (C++)
                 (极致性能，难用)
```

### 与竞品对比的核心优势

| 维度 | mbtgraph | NetworkX | petgraph | JGraphT | LEMON |
|------|:--------:|:--------:|:--------:|:-------:|:-----:|
| **语言** | MoonBit ⭐ | Python | Rust | Java | C++ |
| **多后端** | native+wasm+js ⭐ | Python only | Native | JVM | Native only |
| **wasm 体积** | < 1MB ⭐ | N/A | ~2MB | >50MB | N/A |
| **Trait 层数** | 6 层 ⭐ | 3 层 | 3 层 | 4 层 | 2 层 |
| **存储种类** | 8 种 ⭐ | 4 种 | 5 种 | 6 种 | 3 种 |
| **易用性** | 高 | 极高 | 中 | 中 | 低 |
| **性能** | 高 | 中 | 高 | 中 | 极高 |
| **纯函数** | ✅ 天然支持 ⭐ | ❌ | 部分 | ❌ | ❌ |

**⭐ = mbtgraph 的独特优势**

## 🔄 项目治理原则

### 开发哲学

1. **先做深，再做广** — 每个算法都要做到生产级质量，不追求数量
2. **纯函数优先** — 利用 MoonBit 不可变性，避免副作用
3. **Trait 抽象** — 算法与存储解耦，一套算法适用所有图类型
4. **测试驱动** — 黑盒 + 白盒双轨测试，覆盖率和正确性并重
5. **文档同步** — 代码、测试、文档原子提交，永不分离

### 质量标准

- 所有公开 API 必须有文档注释
- 所有算法必须有复杂度分析和边界情况处理
- 所有模块必须通过 `moon check` 零错误零警告
- 所有变更必须保持向后兼容（直到 v1.0.0 冻结）
