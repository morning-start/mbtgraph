## 1. 背景与问题

### 1.1 当前状态

| 指标 | 数值 |
|------|:----:|
| MEMORY.md 当前行数 | **~542 行** |
| memory/ 目录 | **不存在**（AGENTS.md 设计了但从未创建） |
| 日志混入主文件 | **是**（L401-542 为日志/实现细节） |
| 动态统计数据污染 | **是**（测试数/版本号散布全文） |
| 与其他文档重复 | **是**（Sprint计划 vs TODO.md, 复杂度表 vs algorithms_catalog.md） |

### 1.2 核心问题

1. **两层设计只实现了一半**：AGENTS.md 定义了 `MEMORY.md` + `memory/YYYY-MM-DD.md` 双层结构，但 `memory/` 目录从未创建
2. **职责边界不清**：稳定知识（架构约定）与一次性内容（每日日志/踩坑清单）混杂
3. **动态数据膨胀**：测试数、版本号等高频变化数据散布全文，导致频繁更新但总有过期信息
4. **内容重复**：Sprint 计划与 TODO.md 重复、复杂度表与 algorithms_catalog.md 重复

### 1.3 设计目标

- MEMORY.md 从 ~542 行瘦身至 **~150 行**（减少 72%）
- 建立 `memory/` 日志目录并定义写入规范
- 在 AGENTS.md 中写入完整的记忆系统操作规范
- 消除所有跨文档重复内容

## 2. 方案：严格双层架构

### 2.1 文件结构

```
mbtgraph/
├── MEMORY.md                  # 项目记忆 (稳定, ~150行)
│   ├── §1  项目基础           # 身份/版本/语言/协议/统计摘要(一行)
│   ├── §2  架构约定           # Trait分层 + 存储选型 + 包结构(精简)
│   ├── §3  编码规范摘要       # R1-R7 + Top10陷阱 + 语法规则(精简)
│   ├── §4  关键决策索引       # 仅日期+一句话+链接
│   ├── §5  快速参考           # 泛型函数/Option/Result/Git示例
│   └── §6  工具链             # moon check/test/fmt/build
│
├── memory/                    # 操作日志 (按需增长)
│   ├── 2026-06-03.md          # 当日日志
│   ├── 2026-05-25.md          # 历史日志 (从 MEMORY.md 迁移)
│   └── ...                    # 每工作日一个文件
│
└── AGENTS.md                  # 替换 L338-345 为完整「记忆系统」章节
```

### 2.2 核心原则

**MEMORY.md = 稳定知识库**（低频更新，Agent 启动必读）
**memory/ = 流水日志**（按需写入，当日必读、历史选读）
两者职责零重叠。

## 4. AGENTS.md 记忆系统章节

替换 AGENTS.md 当前 L338-345（原有简略版），新增以下完整章节：

```markdown
## 记忆系统

### 分层架构

| 层级 | 文件 | 性质 | 行数上限 | 职责 |
|------|------|:----:|:--------:|------|
| **项目记忆** | `MEMORY.md` | 稳定知识库 | ~150 行 | 架构约定、决策索引、编码摘要 |
| **操作日志** | `memory/YYYY-MM-DD.md` | 流水日志 | 不限 | 当日操作、实现经验、发布记录 |

### 内容分类速查

**→ 放入 MEMORY.md**:
- 项目身份信息（版本/语言/协议/模块数摘要）
- 架构约定（Trait 分层、存储选型模式、包结构精简版）
- 编码规范摘要（R1-R7 + Top10 陷阱 + 语法规则表）
- 关键决策索引（日期 + 一句话 + 链接；不拷贝全文）

**→ 放入 memory/ 日志**:
- 每日工作记录（做了什么、产出什么、决策过程）
- 一次性实现经验（踩坑清单、依赖顺序、调试过程）
- 版本发布记录（变更清单、发布决策）

**→ 不放记忆系统（已有归属）**:
- 动态测试数/版本统计 → `CHANGELOG.md`
- 算法复杂度详情 → `docs/algorithms_catalog.md`
- Sprint 任务计划 → `docs/TODO.md`
- 详细设计文档 → `docs/design/*.md`
- 开发效率基准 → 不需要持久化

### 写入规则

1. **MEMORY.md 更新时机**（低频）:
   - 新模块完成 → 追加到 §2 包结构 / §4 决策索引
   - 架构变更 → 修改 §2 架构约定段
   - 发现新编码陷阱 → 追加到 §3 编码规范摘要
   
2. **memory/ 日志写入规范**:
   - 文件名格式：`YYYY-MM-DD.md`（使用工作日期）
   - 结构模板：
     ```markdown
     # YYYY-MM-DD 工作日志

     ## 当日目标
     - [ ]

     ## 完成事项
     - [ ] 产出物（文件/代码/文档）

     ## 遇到的问题
     - [ ] 问题描述 + 解决方案

     ## 关键经验
     - [ ] 可复用的经验/踩坑
     ```
   - 每个工作日一个文件，无工作的日子不创建

3. **MEMORY.md 瘦身检查清单**（每次更新后执行）:
   - [ ] 有没有动态统计数据？（应删除或委托 CHANGELOG）
   - [ ] 有没有单次操作记录？（应移至 memory/）
   - [ ] 有没有和其他文档重复的内容？（应删除并引用原文）
   - [ ] 总行数是否接近 200 行？（触发审查）

### 读取流程

```
Agent 启动 → 读 MEMORY.md（必读，~150行快速加载）
            → 读 memory/今日.md（选读，恢复当日上下文）
            → 开始工作
```

- MEMORY.md 是**必读**（每个会话启动时自动加载）
- memory/ 当日日志是**推荐读**（恢复上下文）
- memory/ 历史日志是**按需查**（遇到相关问题时搜索）
```

## 5. 操作流程

### 5.1 Agent 启动时

1. 读取 `MEMORY.md`（~150 行，秒级加载项目全貌）
2. 读取 `memory/YYYY-MM-DD.md`（如有，恢复当日上下文）
3. 开始工作

### 5.2 工作中

| 场景 | 操作 | 写入目标 |
|------|------|---------|
| 遇到架构决策 | 记录决策+原因 | MEMORY.md §4 决策索引 |
| 遇到新编码陷阱 | 记录规则+正确写法 | MEMORY.md §3 规范摘要 |
| 日常开发操作 | 记录做了什么 | memory/当日.md |
| 踩坑/调试经验 | 记录问题和解决方案 | memory/当日.md |

### 5.3 工作结束后

1. 整理 `memory/当日.md`（结构化当日产出）
2. 检查 MEMORY.md 是否需要更新（用瘦身检查清单）
3. memory/ 日志文件可选提交（不强制）

## 8. 风险与缓解

| 风险 | 概率 | 缓解措施 |
|------|:----:|---------|
| 迁移后 Agent 丢失历史上下文 | 中 | memory/ 日志保留完整历史，搜索引擎可检索 |
| MEMORY.md 精简过度丢失有用信息 | 低 | 决策以「索引+链接」方式保留，不删知识点 |
| memory/ 日志无人维护变成垃圾场 | 低 | 明确定义写入模板和场景，不鼓励随意写入 |
