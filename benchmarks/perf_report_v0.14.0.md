## 优化概览

| Task | 优化内容 | 模块 | 收益 |
|:----:|---------|:----:|:----:|
| **A** | 修复 heap `array_pop_last` O(n) → O(1) | 工具函数 | 最短路径 2-5x |
| **B** | CSR Builder 冒泡排序 → 快速排序 | CSR 存储 | 大图构建 10-100x |
| **C** | `neighbors_with_weight` 消除冗余 `get_edge` | Trait + 8 存储 | 最短路径 1.5-3x |
| **D** | Louvain 数据结构重构 + 增益公式修正 | 社区检测 | 10K 节点 5-20x |
| **E** | AdjList 无检查加边 + 批量加边 | 邻接表存储 | 建图 2-3x |
| **F** | CSR 反向索引 (in_neighbors O(V+E)→O(1)) | CSR 存储 | 入边查询加速 |
| **G** | Benchmark 框架 + 基线数据采集 | 基准框架 | 量化度量 |
| **H** | Trait 动态调度/内联分析 | 全局分析 | 编译器优化指引 |

## 关键发现 (Dispatch 分析)

来自 [文档站点 → 调度分析](https://morning-start.github.io/mbtgraph/reference/dispatch-analysis/)：

1. **`node_count` 是最热方法**（82 次，占 GraphReadable 的 37.8%），内联收益最高
2. **前 6 个方法**占总调用的 **95.6%**
3. **GraphWritable** 的 681 次调用全部来自测试，不影响运行时性能
4. **8 个方法从未被使用**，建议清理

### 内联候选 (P0)

| 方法 | 调用次数 | 方法体 | 收益 |
|------|:--------:|:------:|:----:|
| `node_count` | 82 | 1 行 | 每个算法入口调用 |
| `degree` | 14 | 1-3 行 | 循环中高频 |
