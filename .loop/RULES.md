# DocLoop Rules — 三层质量检查清单

> 每个文件必须按顺序通过这三关。**禁止跳过任何检查项。**

---

## 🥇 Correctness（正确性）— 最高优先级

**原则**: 文档里的每一行代码、每一个数字、每一个链接都必须真实可靠。

### 检查项

| # | 检查内容 | 方法 | 不通过处理 |
|---|---------|------|-----------|
| C1 | **代码可编译** | 文档中的 ` ```moonbit ` 代码块调用的函数名必须与 `lib/` 下公开 API 签名一致 | 修改为正确 API |
| C2 | **完全限定名正确** | `@core.Trait::method` 形式的调用必须与 `lib/core/traits.mbt` 中的实际定义匹配 | 修正限定名路径 |
| C3 | **测试/算法/版本号一致** | README 和 site 中提到的测试数、算法数、版本号必须与 `CHANGELOG.md` 和 `moon.pkg` 中一致 | 更新数字 |
| C4 | **超链接不 404** | 所有 `](path)` 和 `<a href>` 指向的文件必须存在 | 修复路径或删除 |
| C5 | **Frontmatter YAML 合法** | `title:` / `description:` 中含 `:`、`&`、`"`、`{` 等特殊字符时必须加引号 | 加双引号 |
| C6 | **路径引用正确** | 引用的 moonbit 包名、文件路径必须真实存在 | 修正路径 |
| C7 | **代码输出与预期一致** | 文档中的"预期输出"应与实际运行结果一致 | 运行验证后修正 |
| C8 | **Trait 层级关系正确** | 描述 Trait 继承关系时（如 GraphReadable → GraphWritable）必须与 `traits.mbt` 实际定义一致 | 修正层级描述 |

### 判断标准

```
✅ 通过: 全部检查项无错误
❌ 不通过: 任一项存在错误 → 标记 BLOCKED，报告给用户
```

## 🥈 Accuracy & Conciseness（准确简洁）— 中等优先级

**原则**: 每段话都有信息量，没有废话，术语一致。

### 检查项

| # | 检查内容 | 通过标准 |
|---|---------|---------|
| A1 | **有明确目标** | 每篇教程开篇有 `🎯 本节目标` 或类似标记说明读者将学到什么 |
| A2 | **术语一致性** | 全库保持统一："Trait" 不混用"特质"、"NodeId" 不写成"node ID"，首次出现英文术语附中文翻译（如"BFS（广度优先搜索）"） |
| A3 | **无空话套话** | 正文不含"众所周知""显而易见""不言而喻""简单来说"等冗余词 |
| A4 | **代码完整性** | 每个代码示例包含完整的 创建→运行→输出 三个步骤，不出现截断的片段 |
| A5 | **段落适中** | 每段不超过 6 行，超过应拆分 |
| A6 | **无重复信息** | 同一知识点不在同一文件内重复出现两次以上 |
| A7 | **示例有上下文** | 代码示例前至少有 1 句话说明"这段代码做什么" |
| A8 | **引用准确** | 引用其他文件时说明具体章节，而非笼统的"见上文"、"详见文档" |

### 判断标准

```
✅ 通过: A1-A8 全部满足 → 进入下一关
⏳ 需优化: 1-3 项不满足 → 修改后重新检查
❌ 不通过: >3 项不满足 → 大幅优化后重检
```

## 🥉 Aesthetics（美观大方）— 优化优先级

**原则**: 排版统一、视觉舒适、不分散注意力。

### 检查项

| # | 检查内容 | 通过标准 |
|---|---------|---------|
| E1 | **代码块语言标注** | 每个 ` ``` ` 后必须跟语言名：`moonbit`、`bash`、`json`、`text`、`python` |
| E2 | **表格格式正确** | 表头用 `| --- | --- |` 分隔，列对齐用 pipe |
| E3 | **中文与英文间距** | 中文和英文之间必须有一个空格，如"使用 BFS 遍历"而非"使用BFS遍历" |
| E4 | **全角/半角标点规范** | 正文用全角标点（，。：；！？）、代码和引用用半角标点（, . : ; ! ?） |
| E5 | **emoji 克制使用** | 每篇文件装饰性 emoji（🎯 ✅ ⭐）不超过 5 个，不使用 emoji 替代正式文字 |
| E6 | **标题不跳级** | Markdown 标题不能跳级，如 `#` → `###` 中间缺 `##` |
| E7 | **代码块内无格式错误** | moonbit 代码块中不出现 Markdown 语法被误解析的情况 |
| E8 | **文件末尾空行** | 每个 .md 文件末尾保留一个空行 |
| E9 | **引用/提示块规范** | 使用 `>` 引用时不超过 3 行，长内容拆为正文 |

### 判断标准

```
✅ 通过: E1-E9 全部满足 → 该文件全部完成
⏳ 需优化: 部分不满足 → 修改后重检
```

## 附：巡查顺序

对一个文件一次循环内的执行顺序：

```
C1 → C2 → C3 → C4 → C5 → C6 → C7 → C8
  ↓ (全部通过)
A1 → A2 → A3 → A4 → A5 → A6 → A7 → A8
  ↓ (全部通过)
E1 → E2 → E3 → E4 → E5 → E6 → E7 → E8 → E9
  ↓ (全部通过)
🎉 该文件完成
```

任何检查项失败 → 修改 → 从该关卡**开头**重新检查（确保修复没引入新问题）。
